# C-Modern Lexical Analyzer

A robust, production-grade Lexical Analyzer (Scanner) built using **Flex**. This project is designed to tokenize the full **ISO C23 standard** (and all preceding standards like C89, C99, C11) for use in a complete Compiler Design pipeline.

---

## 1. Features

* **Comprehensive Standard Support:** Tokenizes all 50+ keywords, including modern additions like `_Generic`, `nullptr`, `bool`, and `_Alignas`.
* **Advanced Numeric Literals:** * **Hexadecimal:** `0x1A2B`
* **Octal:** `0755`
* **Scientific Notation:** `1.2e-10`
* **Type Suffixes:** `100UL`, `3.14f`


* **String & Character Handling:**
* Supports wide-character literals (`L"string"`, `L'c'`).
* Handles escape sequences (`\n`, `\t`, `\\`).
* Detects and reports **Unterminated Strings**.


* **Resilience & Preprocessing:**
* **Preprocessor Protection:** Ignores `#include`, `#define`, and other `#` directives to prevent scanning errors.
* **Line Splicing:** Correctly handles backslash line-joins (`\`).
* **Comment Handling:** Supports both block comments (`/* ... */`) and line comments (`// ...`).


* **Precise Error Reporting:** Tracks line numbers (`lno`) and identifies specific illegal characters or unterminated blocks.

## 2. Token Categories

| Category | Examples |
| --- | --- |
| **Keywords** | `int`, `volatile`, `_Alignas`, `nullptr`, `restrict` |
| **Multi-char Operators** | `++`, `->`, `<<=`, `&&`, `==`, `...` |
| **Single-char Operators** | `=`, `+`, `-`, `*`, `/`, `%`, `&`, ` |
| **Constants** | `1024`, `0xFF`, `3.14f`, `'A'`, `L'B'` |
| **Identifiers** | `_varName`, `function_1`, `counter` |
| **Strings** | `"Hello World"`, `L"Wide String"` |

## 3. Project Structure

* `lexer.l`: The Flex source file containing regular expressions and tokenization rules.
* `parser.y`: The Bison/Yacc file defining the grammar and token bridge (Phase 2).
* `parser.tab.h`: The header file generated by Bison that maps token names to integer IDs.

---

## 4. Build and Execution

### Prerequisites

* **Flex** (Fast Lexical Analyzer Generator)
* **Bison** (GNU Parser Generator)
* **GCC** (GNU Compiler Collection)

### Step-by-Step Compilation

1. **Generate the Parser Header:**
```bash
bison -d parser.y

```


2. **Generate the Lexer Source:**
```bash
flex lexer.l

```


3. **Compile the Executable:**
```bash
gcc lex.yy.c parser.tab.c -o c_scanner

```


4. **Run the Scanner:**
```bash
./c_scanner < input_file.c

```



---

## 5. Error Detection

The lexer is equipped with specialized error rules to catch common syntax mistakes before they reach the parser:

* **Unterminated Comment:** Triggers if a file ends before a `*/` is found.
* **Unterminated String:** Triggers if a newline occurs before a closing `"` is found.
* **Unknown Character:** Reports any character not belonging to the C character set (e.g., `@`, `$`).
